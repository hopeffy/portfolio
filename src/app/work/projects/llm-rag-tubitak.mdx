## Project Overview

An early, hands-on experiment to learn how Retrieval-Augmented Generation (RAG) works, built with **n8n** — a visual workflow-automation tool — soon after it added RAG capabilities. Using TÜBİTAK support-program documents as the knowledge base, the workflow answers questions in natural language by retrieving relevant passages and letting a language model generate grounded answers.

## How It Works

Built entirely as an n8n workflow (no custom backend code):

### 🔍 Document retrieval
- Ingests TÜBİTAK support-program documents and stores them as vector embeddings
- Retrieves the passages most relevant to a user's question via semantic search

### 🤖 Answer generation
- Feeds the retrieved passages to a language model as context
- Generates answers grounded in the source documents

### 🧪 Why I built it
- A practical way to understand the full RAG pipeline — chunking, embeddings, retrieval and grounded generation — using a low-code tool, before building RAG systems in production code

## 🇹🇷 Proje Detayları

RAG'in (Retrieval-Augmented Generation) nasıl çalıştığını öğrenmek için, **n8n** — görsel bir iş akışı otomasyon aracı — henüz RAG özelliklerini eklediği dönemde yapılmış erken bir deneme projesi. TÜBİTAK destek programı belgelerini bilgi tabanı olarak kullanan iş akışı; ilgili bölümleri getirip bir dil modeline dayanaklı yanıt ürettirerek soruları doğal dilde yanıtlar.

### Nasıl Çalışır

Tamamen n8n iş akışı olarak kuruldu (özel backend kodu yok):

### 🔍 Belge Getirme
- TÜBİTAK destek programı belgelerini alıp vektör embedding'leri olarak sakladım
- Kullanıcının sorusuyla en alakalı bölümleri semantik arama ile getirdim

### 🤖 Yanıt Üretimi
- Getirilen bölümleri bağlam olarak dil modeline verdim
- Yanıtları kaynak belgelere dayandırdım

### 🧪 Neden Yaptım
- Üretim koduyla RAG sistemleri geliştirmeden önce; parçalama (chunking), embedding, getirme ve dayanaklı üretim adımlarının tamamını düşük kodlu bir araçla uçtan uca anlamak için pratik bir yol
